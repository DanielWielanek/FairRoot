11 package mangos is an implementation in pure Go of the SP ("Scalable Protocols")
12 protocols. This makes heavy use of go channels, internally, but it can operate
13 on systems that lack support for cgo. It has no external dependencies.
14 
15 The reference implementation of the SP protocols is available as
16 [nanomsg](http://www.nanomsg.org)
17 
18 The design is intended to make it easy to add new transports with almost trivial
19 effort, as well as new topologies ("protocols" in SP terminology.)
20 
21 At present, all of the Req/Rep, Pub/Sub, Pair, Bus, Push/Pull, and
22 Surveyor/Respondent patterns are supported.
23 
24 Additionally, there is an experimental new pattern called STAR available. This
25 pattern is like Bus, except that the messages are delivered not just to
26 immediate peers, but to all members of the topology. Developers must be careful
27 not to create cycles in their network when using this pattern, otherwise
28 infinite loops can occur.
29 
30 Supported transports include TCP, inproc, IPC, Websocket, Websocket/TLS and TLS.
31 Use addresses of the form "tls+tcp://<host>:<port>" to access TLS.
32 Note that ipc:// is not supported on Windows (by either this or the reference
33 implementation.) Forcing the local TCP port in Dial is not supported yet (this
34 is rarely useful).
35 
36 Basic interoperability with nanomsg has been verified (you can do so yourself
37 with nanocat and macat) for all protocols nanomsg supports. Additionally there
38 are a number of projects that use the two products together.

43 ## Installing
44 
45 ### Using *go get*
46 
47  $ go get -u github.com/go-mangos/mangos
48 
49 After this command *mangos* is ready to use. Its source will be in:
50 
51  $GOPATH/src/pkg/github.com/go-mangos/mangos
52 
53 You can use `go get -u -a` to update all installed packages.

61 ## Testing
62 
63 This package supports internal self tests, which can be run in
64 the idiomatic Go way, although it uses a separate test sub-package:
65 
66  $ go test github.com/go-mangos/mangos/test
67 
68 There are also internal benchmarks available:
69 
70  $ go test -bench=. github.com/go-mangos/mangos/test
