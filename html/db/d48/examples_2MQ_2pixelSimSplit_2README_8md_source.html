1 # MC simulations in FairMQ
2 
3 The example shows the possibility to separate the generation of the MC events from their transport in Geant engines.
4 
5 ### FairMQPrimaryGeneratorDevice
6 Generates MC events and sends them. It can also split events into smaller chunks.
7 The size (in the number of particles) of the maximal chunk is specified via chunk-size program option.
8 The default option is to transport whole events (the value is set to 0):
9 ```
10 ./pixel-sim-gen --chunk-size 10
11 ```
12 
13 ### FairMQTransportDevice
14 Receives events (or chunks of them) and transports them through a given geometry.
15 The detector geometry may be specified:
16 - statically: in the executable program that starts the device:
17 ```
18 Pixel* det = new Pixel("PixelDetector", kTRUE);
19  det->SetGeometryFileName("pixel.geo");
20  detArray->Add(det);
21  run->SetDetectorArray (detArray);
22  run->SetStoreTraj (false);
23 ```
24 - dynamically, via the extern C libraries loading mechanism through program options:
25 ```
26 ./pixel-sim-transport --detector-library libTutorial1.dylib --detector-library libPixel.dylib
27 ```
28 The latter option requires implementation of the extern C function ExternCreateDetector, fe.:
29 ```
30 extern "C" void ExternCreateDetector() {
31  FairRunSim* run = FairRunSim::Instance();
32  Pixel* det = new Pixel("PixelDetector", kTRUE);
33  det->SetGeometryFileName("pixel.geo");
34  run->AddModule(det);
35 }
36 ```
37 
38 ### FairMQChunkMerger
39 Merges chunks back into full events.

41 # Running
42 
43 The generator and transporter may run in one of two modes:
44 - request-reply: the generator waits for the requests from the transporter(s) before
45 generating new events. This mode is optimal for large detector setups, where transporter
46 CPU cost is large and the number of simulated events is typically low.
47 - push-pull: the generator pushes the events, and the transporter(s) is pulling. This running mode
48 is useful for small detector setups with small CPU cost, where plenty of events are generated.
