3 ## The FairRoot framework
4 A simulation, reconstruction and analysis framework that is based on the ROOT system.
5 The user can create simulated data and/or perform analysis with the same framework. Geant3 and Geant4 transport engines are supported, however the user code that creates simulated data do not depend on a particular monte carlo engine. The framework delivers base classes which enable the users to construct their detectors and /or analysis tasks in a simple way, it also delivers some general functionality like track visualization. Moreover an interface for reading magnetic field maps is also implemented.

17 ## Using the Project template
18 
19 FairRoot deliver meanwhile a project template that can be used as a starting point for anybody who would like to build simulation and reconstruction on FairRoot. The project Template is in the [FairRoot/template/project_template](https://github.com/FairRootGroup/FairRoot/tree/dev/templates/project_template) directory
20 
21 The template demonstrate and implement the following:
22 
23 1. General structure of the software (cake config files, VMC/Geant configurations, etc ..)
24 2. Example detector with sensitive and passive volumes (NewDetector) and data class
25 3. Particle Stack for Geant3/4 with filtering infrastructure
26 4. Event generators (Pathia6,8) more are available directly from FairRoot
27 5. Passive component implementation (Magnet Yoke, Beam Pipe)
28 6. Track visualisation tool (Event display)
29 7. A rename script which replace all the generic names to user defined ones

31 ### Optional change of output tree name
32 
33 By default, the name of the tree in output ROOT file is "cbmsim". In case you want to change it for your Project, you need to create "config" folder inside of top source directory of the Project and place there "rootmanager.dat" text file with following single line: "treename=name_you_chose".
34 
35 ```bash
36 cd PROJECT_TOP_DIR
37 mkdir config
38 echo 'treename=name_you_chose' > config/rootmanager.dat
39 ```

42 ### Step by Step installation
43 
44 1. Install [FairSoft](https://github.com/FairRootGroup/FairSoft)
45 
46 2. Install [FairRoot](http://fairroot.gsi.de/?q=node/82)
47 
48  ```bash
49  # Set the shell variable SIMPATH to the FairSoft installation directory
50  export SIMPATH=~/fair_install/FairSoftInst
51  [setenv SIMPATH ~/fair_install/FairSoftInst]
52 
53  cd ~/fair_install
54  git clone -b dev https://github.com/FairRootGroup/FairRoot.git
55  cd FairRoot
56  mkdir build
57  cd build
58  cmake -DCMAKE_INSTALL_PREFIX="~/fair_install/FairRootInst" ..
59  make
60  make install
61  ```
62 
63  To run the tests do:
64 
65  ```bash
66  # To run test: make new shell, do not define SIMPATH
67  cd ~/fair_install/FairRoot/build
68  make test
69  ```
70 
71 3. Install the template:
72 
73  you need to copy the [project template](https://github.com/FairRootGroup/FairRoot/tree/dev/templates/project_template) to you own directory
74 
75  ```bash
76  # Set the shell variable FAIRROOTPATH to the FairRoot installation directory
77  export FAIRROOTPATH=~/fair_install/FairRootInst
78  [setenv FAIRROOTPATH ~/fair_install/FairRootInst]
79 
80  cd ~/fair_install
81  cp -rf FairRoot/templates/project_template MyTest
82  cd MyTest
83  ./rename.sh MyExperiment PREFIX MyDetector
84 
85  # Please call the script with three parameters. The first one is the
86  # name of the project. The second one is the prefix in front of
87  # the class names. and some directories. So this second parameter
88  # shouldn't be to long. The third parameter is the name of the
89  # detector you want to implement.
90  # As an example the if you want to create a project for the Panda
91  # experiment and you want to implement a Straw Tube Tracker (stt)
92  # you would call the script in the following way.
93  # ./rename.sh Panda Pnd stt
94 
95  cd ..
96  mkdir build
97  cd build
98  cmake ../MyExperiment
99  make
100  . config.sh [or source config.csh]
101  ```
102 
103  Now you can for example simulate some events and run the event display:
104 
105  ```bash
106  root -q ../MyExperiment/macro/run_sim.C
107  root ../MyExperiment/macro/eventDisplay.C
108  // Click on "FairEventManager" (in the top-left pane)
109  // Click on the "Info" tab (on top of the bottom-left pane)
110  // Increase the "Current Event" to >0 to see the events
111  root [1] .q
112  ```
